Summit Data Migrate Document for API

Goals

The Goals data import only comes from Summit, and there is no relevant data in Lattice, so it does not need to be considered.

Data Mapping

First, we need to perform data mapping between Summit and Workday systems. There are some structural differences between these two systems:

  • In Summit: Expectations are cycle-based, where one Cycle can contain multiple Expectations in a parent-child relationship
  • In Workday: Each Goal is an independent object with its own attributes

Therefore, we need to decompose the Expectations from Summit and map their attributes accordingly. During this process, we found that some Goal attributes don’t exist in Summit. After discussing with Keqian and Vivian, we knowed that these missing values are optional, which is acceptable for our migration, so we can proceed with the mapping.

Mapping Table

The mapping relationship is as follows (tentative required fields):

Summit Workday Additional info
Expectation Goal
title(expectation) Name
content(expectation) Description Workday support richtext
category(expectation) Goal_Category

Note: The mapping fields can be modified according to business requirements

API Selection

In Workday’s SOAP API, we identified an API called ‘Manage Goals’ that aligns with our requirements. This API provides functionality for creating, updating, and deleting goals.

API Documentation: https://community.workday.com/sites/default/files/file-hosting/productionapi/Performance_Management/v42.2/Manage_Goals.html

API Usage

Requirements

  • API Client(Function area: Performance Enablement)
  • Integration system user
  • Security group

Setup Steps

We need share the ‘Manage Goals’ Initiating Action access for the security group, and then add the integration system user to the security group.

API Request Example

We only need to fill in the following fields in the request body:

  • Worker_Reference: The employee ID of the user who is creating the goal
  • Name: The title of the goal
  • Description: The content of the goal
  • Goal_Category_Reference: The category of the goal
<env:Envelope
	xmlns:env="http://schemas.xmlsoap.org/soap/envelope/"
	xmlns:bsvc="urn:com.workday/bsvc">
	<env:Header>
		<bsvc:Workday_Common_Header></bsvc:Workday_Common_Header>
	</env:Header>
	<env:Body>
		<bsvc:Manage_Goals_Request
			xmlns:bsvc="urn:com.workday/bsvc" bsvc:Add_Only="false" bsvc:version="v42.2">
			<bsvc:Manage_Goals_Data>
				<bsvc:Worker_Reference bsvc:Descriptor="string">
					<bsvc:ID bsvc:type="Employee_ID">39178</bsvc:ID>
					<!-- 修改ID类型 -->
				</bsvc:Worker_Reference>
				<bsvc:Worker_Goal_Detail_Data>
					<bsvc:Name>Develop my leadership skills</bsvc:Name>
					<bsvc:Description>
						<![CDATA[Expectation:
                        The expectation is to develop my leadership skills and take more responsibilities.

                        Actions to be taken:
                        - Coordinate within the team to discuss the things or issues faced by most of the team members.
                        - Provide KT to the team members if I learnt something new.
                        - Show commitment and dedication to the team goals.
                        - Work on time management and prioritization of tasks.
                        - Improve communication skill.

                        Know how it is done:
                        - Will be gaining more responsibility within the team.
                        - Noticing a change in collaboration.
                        - Getting positive feedbacks from peers and client.
                        ]]>
					</bsvc:Description>
					<bsvc:Goal_Category_Reference bsvc:Descriptor="string">
						<bsvc:ID bsvc:type="Goal_Category_ID">GOAL_CATEGORY-6-1</bsvc:ID>
						<bsvc:ID bsvc:type="Goal_Category_ID">GOAL_CATEGORY-6-2</bsvc:ID>
					</bsvc:Goal_Category_Reference>
					<bsvc:Goal_Archived>false</bsvc:Goal_Archived>
				</bsvc:Worker_Goal_Detail_Data>
			</bsvc:Manage_Goals_Data>
		</bsvc:Manage_Goals_Request>
	</env:Body>
</env:Envelope>

Solution

First, use a database connection tool to access the Summit database system, execute SQL queries to extract required field information, including but not limited to expectation titles, content and other key data. Export the query results and save them in spreadsheet format.

Then develop an automation script that will execute the following steps:

  1. Call Workday authentication API to obtain access token
  2. Read data from the spreadsheet and convert it to Workday API compliant request format
  3. Batch call Workday API interfaces to complete data migration

Through this process, efficient and accurate data migration from Summit system to Workday platform can be achieved.
