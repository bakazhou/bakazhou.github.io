如何使用Integration System增强原生Web Service?

Bakazhou Blog
2024-11-30

前言

前一篇Workday集成指南中，我们介绍了Workday集成服务的基本概念和使用方法。在其中我们多次提到了Integration System，因为之前此功能依然处于探索阶段，所以没有深入介绍。但在近日的探索中发现并成功应用了其中的一项功能：Integration System可以用于增强Workday原生的Web Service，在此我们便来详细介绍下这项功能以及如何运用。

概述

Integration System是Workday提供的一项功能，它允许用户通过配置和使用Integration System来增强Workday原生的Web Service。Integration System可以用于以下场景：

  • 在Workday中创建、更新或删除数据
  • 在Workday中执行复杂的业务逻辑
  • 在Workday中进行数据转换和映射
  • 在Workday中进行数据验证和错误处理
  • 在Workday中进行数据分析和报告

应用

创建配置步骤

创建Integration System

使用以下Task来创建Integration System：

Create Integration System

完成以上步骤后，我们便拥有了一个Integration System，名称为Tianle_IS。

注意事项

  • 创建Integration System时，Template是必选项，这些模板定义了Integration System的配置方式，包括数据源、数据转换、数据验证等。它们都来自于Workday的官方配置，我们需要根据实际业务需求选择合适的模板。
  • Integration Tags是Workday提供的一项功能，它允许用户为Integration System添加标签，以便于管理和分类。标签可以用于标识Integration System的用途、功能、业务领域等，从而方便用户进行查找和使用。但这是一项可选项，可以根据自己的需求选择是否添加。
  • 在此我们假设业务场景为: 在获取Position数据时，业务需要Position相关的Cost Center信息。因此我们选择的Template为Core Connector: Positions。

增强Get Positions

将Tianle_IS在请求体中进行引用，如下所示：

<env:Envelope xmlns:env="http://schemas.xmlsoap.org/soap/envelope/" xmlns:bsvc="urn:com.workday/bsvc">
    <env:Header>
        <bsvc:Workday_Common_Header>
            <!-- 这里可以包含认证信息 -->
        </bsvc:Workday_Common_Header>
    </env:Header>
    <env:Body>
        <bsvc:Get_Positions_Request bsvc:version="v42.2">
            <bsvc:Request_Criteria>
                <bsvc:Field_And_Parameter_Criteria_Data>
                    <bsvc:Provider_Reference>
                        <bsvc:ID bsvc:type="Integration_System_ID">Tianle_IS</bsvc:ID>
